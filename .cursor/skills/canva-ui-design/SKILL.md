# Canva UI Design Principles

Apply these principles when generating UI mockups, designing screens, creating brand assets, or writing image prompts for UI visuals.

## Typography Hierarchy

Use **3 levels**:

| Level | Purpose | Size (typical) |
|-------|---------|----------------|
| 1 | Headline; captures attention | 24–32px |
| 2 | Section headers, subheads, pull quotes | 16–20px |
| 3 | Body, details, fine print | 14–16px |

- **Max 2 font families** per design
- **Pair complex with simple**: e.g. serif + sans-serif
- **Emphasis**: Use bold or larger size; avoid underlining

## Visual Hierarchy

- **Size**: Larger elements draw attention first
- **Color/contrast**: Warm or bright colors draw attention; use sparingly
- **Spacing**: White space separates sections; proximity groups related items
- **Focal point**: Identify primary, secondary, and endpoint; align layout to that flow

## Layout and Composition

- **Rule of Thirds**: Place key elements at grid intersections (divide into 3×3)
- **Rule of Odds**: Odd numbers (1, 3, 5) often work better than even
- **Grids**: 12-column for web; consistent margins and gutters
- **F-pattern**: Text-heavy layouts; users scan top-left first, then down
- **Z-pattern**: Less dense; landing pages, calls to action
- **Golden ratio** (~1.618): Balanced proportions when relevant

## Color

| Type | Colors | Mood |
|------|--------|------|
| Warm | Red, orange, yellow | Energy, enthusiasm |
| Cool | Blue, green, purple | Calm, trust |
| Neutral | Black, white, gray | Pair with warm or cool |

- Use **contrast** (different values/saturations) to create hierarchy
- Use color **sparingly**; avoid overloading

## White Space

- Avoid crowding; let elements breathe
- Scale down elements to create negative space
- **Proximity**: Related items closer, unrelated farther
- Do not feel compelled to fill every space
